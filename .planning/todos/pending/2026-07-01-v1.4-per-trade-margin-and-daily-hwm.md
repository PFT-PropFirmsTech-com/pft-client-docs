Client's new request (comment 2026-07-01 09:49): used margin is only visible while a trade is OPEN; once closed, neither client nor TC has a record of the max used margin the trade reached (unless computed manually from history). They want:
1. **Per-trade max used margin** — record the highest used margin each trade reached while open, and display it in the client's **trading history** on the dashboard (so closed trades keep the record).
2. **Daily Used-Margin High-Water-Mark** — the highest used margin reached at any point during a trading DAY. When a new day's HWM is set, that day updates; previous days keep THEIR day's HWM (a per-day historical series, NOT just one all-time peak).
3. Both surfaces: client dashboard + Trading Cult admin dashboard.

## Solution (shape — needs research at plan time)

Builds on Phase 7 groundwork (`ruleStateService` already computes `currentMarginUsedPercent` per tick; `peakMarginUsedPercent` Math.max ratchet already persists all-time peak on `accountrulestates`).

- **Per-trade max margin (rule-checker → backend → dashboard):** the tricky part is capturing the peak margin a SPECIFIC position consumed while open, then persisting it onto that trade's closed record (`tradehistories`). RESEARCH: does the rule-checker tick loop have per-position margin (MT5 position-level margin), or only account-level? If only account-level, per-trade attribution needs design. Then surface the field in the dashboard trade-history table (client + admin) — new column.
- **Daily HWM series (rule-checker → backend → dashboard):** per-day bucket (date → max margin used %), written per tick with daily rollover — MIRROR the `dailystartingbalances` rollover pattern. Store as an array on `accountrulestates` (e.g. `dailyMarginHwm: [{date, pct}]`) or a new `dailymarginhwm` collection. Each day frozen once passed; today's updates on new highs. Surface as a per-day view/chart on client + admin.
- Spans pft-rule-checker (capture + persist — the hard part) + pft-backend (response fields, via the same accountStatistics/trade-history read paths Phase 7 used) + pft-dashboard (trade-history column + daily HWM UI).
- Reference: v1.2 Phase 7 SUMMARY + `reference_...` margin decisions in PROJECT.md (margin = `margin/equity*100`, NOT MT5 marginLevel; peak all-time monotonic, never reset on daily/EOD/payout). NO retroactive backfill (same as Phase 7 — populates from deploy forward).
