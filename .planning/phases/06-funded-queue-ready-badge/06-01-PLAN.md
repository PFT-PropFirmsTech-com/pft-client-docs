    Behavior (verbatim from `06-RESEARCH.md` §Code Examples, do not deviate):
    ```ts
    getReadyForApprovalCount: async (): Promise<{ count: number; manualApprovalEnabled: boolean }> => {
      const manualApprovalEnabled = await FundedQueueSettingsService.isManualApprovalEnabled();
      if (!manualApprovalEnabled) {
        // Brand auto-processes; sidebar must stay dark.
        return { count: 0, manualApprovalEnabled: false };
      }

      const pendingUserIds = await FundedProgressionQueue.distinct("userId", { status: "pending" });
      if (!pendingUserIds.length) return { count: 0, manualApprovalEnabled: true };

      const { Kyc } = await import("../Kyc/kyc.model");
      const { Contract } = await import("../Contracts/contracts.model");
      const [approvedKycs, approvedContracts] = await Promise.all([
        Kyc.find({ userId: { $in: pendingUserIds }, status: "approved" }).select("userId").lean(),
        Contract.find({
          userId: { $in: pendingUserIds },
          status: "approved",
          fileType: "contract",
        }).select("userId").lean(),
      ]);

      const kycReady = new Set(approvedKycs.map((d) => String(d.userId)));
      const contractReady = new Set(approvedContracts.map((d) => String(d.userId)));
      const readyUserIds = pendingUserIds.filter(
        (id) => kycReady.has(String(id)) && contractReady.has(String(id)),
      );
      if (!readyUserIds.length) return { count: 0, manualApprovalEnabled: true };

      // Count pending entries (not users) — a single user could have >1 funded account pending.
      const count = await FundedProgressionQueue.countDocuments({
        status: "pending",
        userId: { $in: readyUserIds },
      });
      return { count, manualApprovalEnabled: true };
    },
    ```

    Critical constraints (NON-NEGOTIABLE per research):
    - MUST live-compute via `Kyc.find` + `Contract.find`. Do NOT count by stored `kycApproved` / `contractApproved` / `reason: "manual_approval_pending"` fields — those lag the 5-min `scanAndProcessReady` cron (research §Pitfall 1 confirmed empirically against TC live DB: 0 stored matches despite a UI row).
    - MUST short-circuit `{ count: 0 }` when `manualApprovalEnabled === false` (research §Pitfall 3; brands without manual approval auto-process and the badge would blink on rows about to vanish).
    - MUST use `status: "pending"` ONLY — NOT `{ $in: ["pending", "processing"] }` (research §Pitfall 2; `processing` is in-flight cron work, `failed` has its own retry UI).
    - Use the exact same import paths and accessor (`FundedQueueSettingsService.isManualApprovalEnabled`) as `getMyPending` already uses in `fundedProgressionQueue.controller.ts` lines 46-58. If `FundedQueueSettingsService` is not already imported at the top of the service file, add the import — match how it's imported elsewhere in the module (the controller already shows the path).
    - Contract filter MUST include `fileType: "contract"` — Contract collection holds multiple file types; only the signed contract counts.
