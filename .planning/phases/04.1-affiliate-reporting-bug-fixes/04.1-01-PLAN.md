    ## Bug 3 — Product `twoStep $NaN` → parseAccountSize helper

    Same file (`AffiliatesContainer.tsx`), two sites of the `product()` helper (CSV builder around line 178; table render around line 743).

    Current shape (both sites):

    const size = r.payment?.accountSize

    Add a module-scope helper near the other utilities (above `AffiliatesContainer` component, alongside `PurchaseReportTable` if that's where it sits — match existing module-scope pattern):

    ```typescript
    // Mirror of pft-backend src/app/modules/Admin/Program/accountSize.utils.ts parseAccountSizeValue.
    // Accepts "5k" / "10K" / "500k" / "5000" / "1M" → numeric USD value.
    // Returns null if unparseable so callers can fall back to the raw string.
    function parseAccountSize(input: unknown): number | null {
      if (input == null) return null;
      const s = String(input).trim();
      if (!s) return null;
      const match = s.match(/^([\d.]+)\s*([kKmM]?)/);
      if (!match) return null;
      const base = parseFloat(match[1]);
      if (!Number.isFinite(base)) return null;
      const suffix = match[2]?.toLowerCase();
      const multiplier = suffix === "k" ? 1e3 : suffix === "m" ? 1e6 : 1;
      return base * multiplier;
    }
    ```

    Then in BOTH `product()` sites, replace the `size` derivation:
    ```typescript
    const rawSize = r.payment?.accountSize;
    const parsed = parseAccountSize(rawSize);
    const size = parsed != null
      ? `$${parsed.toLocaleString("en-US")}`
      : rawSize
        ? `$${rawSize}` // fall back to raw string (e.g. "$5k") if regex misses, NEVER "$NaN"
        : "";
    ```
