Build the `partnerPostback` GET adapter (CRM-07) and register it. The adapter reads the per-brand `partnerPostback` config (from 12-01), applies the skip-guards + the FTD once-per-user gate, substitutes `{clickid}`/`{goal}`/`{payout}`/`{currency}` macros (encoding the clickid exactly once), fires a fire-and-forget native `fetch` GET with a 15s timeout and one bounded retry on partner 5xx, and returns an `IDispatchResult` (never throws). The FTD gate here (`isFirstPurchase === true`) is the single source of the "conversion fires once per acquired user" guarantee (Phase 11 fires purchase events on EVERY purchase for Meta/GA4/Klaviyo; only the partner conversion is gated). `usdAmount` (payload.value, set by Phase 11's 982ba9a1) guards the JPY-billed-as-USD bug class.

Create a NEW adapter file. Mirror the STRUCTURE of `conversion-webhook.ts` (imports of `IDestinationAdapter`/`IDispatchContext`/`IDispatchResult` from `./base`, `EventName`/`ITrackingEventPayload` from `../tracking.interface`, `logger` from `../../../utils/logger`; export a `const partnerPostbackAdapter: IDestinationAdapter = { name: "partnerPostback", async send(payload, ctx) {...} }`). Do NOT reuse its POST/JSON/HMAC body ā€” this is GET + URL macros.

`send(payload, ctx)` logic, IN ORDER (each early-return builds a result object with `destination: "partnerPostback"`):

1. `const cfg = ctx.settings.destinations.partnerPostback;`
   `if (!cfg?.enabled) return { destination: "partnerPostback", status: "skipped", error: "disabled" };`
   (Defensive ā€” dispatcher already gates on enabled via destinationAcceptsEvent, matches conversion-webhook.ts.)

2. `if (!payload.partnerClickId) return { destination: "partnerPostback", status: "skipped", error: "no partnerClickId" };`
   (No clickid = nothing to attribute.)

3. Select goal + template by `payload.eventName`:
   - `signup_completed` ā†’ `goal = "registration"`, `template = cfg.registrationUrl`, `payout = ""` (no payout on registration).
   - `purchase_completed` OR `pap_payment_completed` ā†’ **FTD GATE FIRST**: `if (payload.isFirstPurchase !== true) return { destination: "partnerPostback", status: "skipped", error: "not first purchase" };` THEN `goal = "conversion"`, `template = cfg.conversionUrl`, `payout = String(payload.value ?? "")` (payload.value === usdAmount, set by Phase 11 982ba9a1), currency = "USD".
   - default (any other eventName) ā†’ `return { destination: "partnerPostback", status: "skipped", error: "event not mapped" };` (should not happen ā€” matrix only routes the three, but be defensive).

4. `if (!template) return { destination: "partnerPostback", status: "skipped", error: "empty url template" };`

5. Macro substitution ā€” build the final URL by replacing placeholders present in the template (only substitute what's there; the partner's `?clickid={clickid}&goal={goal}&payout={payout}` uses these names):
   ```ts
   const encodedClickId = encodeURIComponent(payload.partnerClickId);
   const currency = "USD";
   const url = template
     .replaceAll("{clickid}", encodedClickId)
     .replaceAll("{goal}", encodeURIComponent(goal))
     .replaceAll("{payout}", encodeURIComponent(payout))
     .replaceAll("{currency}", encodeURIComponent(currency));
   ```
   Encode the clickid EXACTLY ONCE here ā€” Phase 10/11 kept it byte-identical raw; this is the single encode point (locked contract). Use `.replaceAll` (Node 18+, available; no new deps).

6. Fire GET with timeout + ONE bounded retry on 5xx. No new deps ā€” native `fetch` + `AbortSignal.timeout`:
   ```ts
   const doGet = async (): Promise<Response> =>
     fetch(url, { method: "GET", signal: AbortSignal.timeout(15000) });
   try {
     let res = await doGet();
     if (res.status >= 500 && res.status < 600) {
       // one bounded retry on partner 5xx
       res = await doGet();
     }
     if (res.ok || (res.status >= 300 && res.status < 400)) {
       return { destination: "partnerPostback", status: "sent", responseMeta: { httpStatus: res.status } };
     }
     return { destination: "partnerPostback", status: "failed", error: `HTTP ${res.status}`, responseMeta: { httpStatus: res.status } };
   } catch (err: unknown) {
     const message = err instanceof Error ? err.message : String(err);
     logger.warn(`partner-postback adapter failed for ${payload.eventName}: ${message}`);
     return { destination: "partnerPostback", status: "failed", error: message };
   }
   ```
   NEVER throw ā€” network error/timeout ā†’ `status: "failed"` (dispatcher's markFailed writes the log row). 2xx/3xx ā†’ "sent".

Add a top-of-file doc comment: GET S2S postback, macro-substituted, fire-and-forget (dispatcher calls it inside Promise.allSettled), FTD gate for conversions, encode-once contract.
