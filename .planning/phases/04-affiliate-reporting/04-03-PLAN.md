The change adds a second step between (1) and (2):

**After fetching payments and before building headers**, add the bulk commission fetch:

```typescript
// Bulk-fetch commissions for all payments in one call
const orderIds = payments.map((p) => p._id);
let commByOrder: Record<string, any[]> = {};
if (orderIds.length > 0) {
  try {
    const commUrl = ENDPOINTS.admin.affiliate.bulkCommissionsByOrders;
    const { data: commResponse } = await apiClient.post(commUrl, { orderIds }, { timeout: 60000 });
    commByOrder = commResponse?.data || {};
  } catch {
    // Commission fetch failed — export continues with empty affiliate columns
    commByOrder = {};
  }
}
```

NOTE: Use `apiClient.post` (POST body) not `apiClient.get` with query string — the URL would exceed browser limits with 2000+ payment IDs.

**Extend the headers array** (currently ends at `"Program Assigned"`) by appending 6 more headers AFTER the existing 19:

```typescript
"Commission Rate (%)",
"Commission Amount (USD)",
"Commission Currency",
"Affiliate User ID",
"Affiliate Name",
"Affiliate Email",
```

**In the rows map**, after the existing `escapeField(p.programAssigned ? "Yes" : "No")` field, add 6 more fields:

```typescript
// Commission columns — use lowest-tier entry for MLM (multiple entries per payment)
const commissions: any[] = commByOrder[p._id] || [];
const c = commissions.length > 0
  ? commissions.sort((a: any, b: any) => (a.tier ?? 1) - (b.tier ?? 1))[0]
  : null;
escapeField(c?.rate ?? ""),
```
